
# 기능 요구사항

## 기본 설정
- 8개의 카테고리가 존재한다 (상의, 아우터,바지, 스니커즈, 가방, 모자, 양말, 액세서리)
- 9개의 브랜드가 존재하고 각 브랜드는 1개의 카테고리 상품을 가지고 있다.
- 각 브랜드별 상품의 가격은 각각 다른 가격을 가지고 있다.
- 요구사항은 총 4개이며 각 요구사항에 만족하는 API를 제공해야한다.
- API는 JSON 형태로 응답해야하며 실패시 실패값과 실패 사유를 전달해야한다.

## 요구사항 1
- 카테고리별 최저가격인 브랜드와 가격을 조회하고 총액이 얼마인지 확인할 수 있어야한다.
- 응답은 JSON 형태로 제공되며 총액과 상품정보를 제공한다
``` json
// ex)
{
    "총액": 100000,
    "상품": [
        {
            "카테고리": "상의",
            "브랜드": "A",
            "가격": 10000
        },
        {
            "카테고리": "아우터",
            "브랜드": "B",
            "가격": 20000
        },
        ...
    ]
}
```
## 요구사항 2
- API 요청시 단일 브랜드로 모든 상품을 구매할때 최저가격에 판매하는 브랜드와 총액을 응답해야한다.
``` json
// ex)
{
    "최저가 : {
        "브랜드": "A",
        "카테고리" : [
            {
                "카테고리" : "상의",
                "가격" : "10000"
            },
            ...
        ],
        "총액": 100000
    }
}
```

## 요구사항 3
- 카테고리 이름으로 API 요청 시 카테고리, 최저, 최고 가격 브랜드와 상품 가격을 응답한다
``` json
{
    "카테고리" : "상의",
    "최저가" : {
        "브랜드" : "A",
        "가격" : 10000
    },
    "최고가" : {
        "브랜드" : "B",
        "가격" : 20000
    }
}
```

## 요구사항 4
- 브랜드 및 상품을 추가 / 업데이트 / 삭제하는 API를 제공한다


# 기능 구현
## 도메인 정리
- 카테고리
    - 이름
- 브랜드
    - 이름
- 상품
    - 카테고리
    - 브랜드 

## 참고사항
- 데이터를 삭제할때 soft delete를 사용하도록 처리하였으며 삭제된 데이터는 조회되지 않도록 처리하였습니다.
- 매번 조회시 삭제된 데이터를 제외하는 조건을 넣는 방식이 해당 프로젝트에서는 불필요하다고 판단하여 SQLRestriction을 사용하였습니다.
- 삭제된 데이터를 조회해야하는 요구사항이 생길경우에는 관리자 로직을 별도로 구현하거나 데이터 조회시 삭제된 데이터 제외 조건을 넣는 방식으로 변경이 필요합니다.
